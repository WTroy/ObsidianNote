哈希表在使⽤层⾯上可以理解为⼀种集合结构
如果只有 key ，没有伴随数据 value ，可以使⽤ HashSet 结构
如果有 key ，也有对应的 value ，可以使⽤ HashMap 结构

使⽤哈希表增删改查的操作，可以认为时间复杂度是 O(1), 但是常数时间⽐较⼤
放⼊哈希表的东⻄，如果是基础类型，内容按值传递，内存占⽤就是这个东⻄的⼤⼩
放⼊哈希表的东⻄，如果不是基础类型，内部按引⽤传递，内存占⽤是这个东⻄内存地址的⼤⼩
有序表在使⽤层⾯上可以理解为⼀种集合结构
如果只有 key ，没有伴随的 value 值，可以使⽤ TreeSet 结构
如果有 key ，也有伴随的 value 值，可以使⽤ TreeMap 结构
有⽆伴随 value 值是 TreeSet 和 TreeMap 的唯⼀区别，底层实际结构是⼀回事
有序表和哈希表的区别：有序表把 key 按照顺序组织起来，⽽哈希表完全不组织
红⿊树， AVL 树， size-balance-tree 和跳表等都属于有序表结构，只是底层具体的实现不同
放⼊有序表的东⻄，如果是基础类型，内容按值传递，内存占⽤就是这个东⻄的⼤⼩
放⼊有序表的东⻄，如果不是基础类型，内部按引⽤传递，内存占⽤是这个东⻄内存地址的⼤⼩
不管是什么样的底层实现，只要是有序表，都有以下固定的基本功能和固定的时间复杂度
