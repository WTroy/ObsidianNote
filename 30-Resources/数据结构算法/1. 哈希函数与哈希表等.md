哈希表的实现：
按找 hash 表初始化的⻓度，将 key 值算出 hash 值，然后取模，放⼊哈希表对应位置， value 是⼀个单链表。
哈希表在使⽤过程中不可避免的会有扩容动作，假设哈希表初始化只有 2 个 key 值，加⼊增加 N 个 str ，则哈希表需要扩容
logN 次，每次扩容的代价是 O(N) ，所以如果数据量为 N 的哈希表，总的扩容代价为 O(N*logN) 。
每次增加⼀个值到哈希表的实际代价为： NlogN/N, 当我们初始化哈希表的 key 值⽐较⼤的时候，增加⼀个值的代价约等
于 O(1), 哈希表有增有减，所以也会相应减少扩容次数。所以哈希表在使⽤上可以认为是 O(1), 但是理论上他仍然是
O(logN).

题⽬四：
详解⼀致性哈希原理：
哈希的 key 值选择：选择分布分散的值，保证⾼频，中频，低频都有数量，不要选种类很少的值，⽐如：性别。
经典哈希 \ 模算法实现的分布式或者负载，在增删服务器的时候会导致全量的重置。
⼀致性哈希：
以 MD5 举例⼦： MD5 计算结果在 0~2^64-1 之间，并把它想象成⼀个环。如果有三台服务器，通过他的唯⼀标识（⽐如
hostname ）计算出 MD5 值，则这三个服务器在这个环中，其他数据进来，通过 MD5 计算，按照值的⼤⼩在这个环上顺
时针寻找，离得最近的节点即为这个数据的负载点或者存储点。
当需要新增或者删除服务器，只会影响顺时针⽅向上的⼀部分数据，不会导致全量更新。
问题：

1. 服务器⽐较少时，不能够保证负载平均负载。
2. 新增或者删除服务器时，会导致负载不均衡。
