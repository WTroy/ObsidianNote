事务（ Transaction ）：
作为⼀个单独单元的⼀个或者多个 sql 语句组成。这个单元中的每个 sql 语句是相互依赖的，⽽且单元作为⼀个整体是不
可分割的。如果单元中的⼀个语句不能成功的完成，整个单元就会回滚，所有影响到的数据库将返回事务开始以前的状
态。因此，只有事务中所有语句都被成功的执⾏才能说这个事务被成功的执⾏。

Mysql 事务的基本概念
⼀   Mysql 事务处理的基本概念 ;
原⼦性 Atomicity ：意味着每个事务都必须被认为是⼀个不可分割的单元。
⼀致性 Consistency ：不管事务是完全成功还是中途失败，事务必须使系统处于⼀致的状态。
孤⽴性 Isolation ：每个事务在它⾃⼰的空间发⽣，和其他发⽣在系统中的事务隔离，⽽且事务的结果只有在它完全被
执⾏时才能看到。
持久性 Durability ：持久性是指即使系统奔溃，⼀个提交的事务仍然存在。

Mysql 事务隔离级别的概念和序列化隔离
serializable ：顺序操作  ，所有开启了事务的 session 同时只能有⼀个进⾏更新操作
repeatable read （可重复读）：会有幻读问题：当两个事务同时查询， A session 已经查询到数据，然后 B session
修改了数据并提交， A session 如果只做多次查询，那查出的数据⼀致；但是，如果有更新操作然后再次查询，虽然
在同⼀个事务中，但是查询的两次数据不同。
read commit （提交读）：会导致不可重复读，如果当前查询到的数据被另⼀个事务修改并提交，会导致再次查询的
数据和上⼀次不⼀致。
read uncommitted （未提交读）：会导致脏读，如果另⼀个事务修改了数据但是并未提交，修改的数据就会被别的
事务读取到，导致读到不正确的数据。
